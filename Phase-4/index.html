    Phase 4

    Erstellung eines Projekt-Wikis

    Erledigt

    In der Phase 4 wurde das hier zu sehende Wiki erstellt. MKDocs wurde dabei auf Github eingebunden. Die Darstellung erfolgt mit der Markup-Sprache Markdown erstellt, die es ermöglicht, Formatierungen wie Überschriften, Absätze, Listen, Links, Bilder und andere Elemente zu erstellen. Zudem lassen sich verschiedene Themes einbinden, womit das Layout der Seite verändert werden kann. Aber auch andere Sprachen wie Java-Script oder HTML lassen sich über Markdown per MKDocs einbinden.

    Wie lässt sich MKDocs einbinden?

    1. Anforderungen

    MKdocs lässt sich über Github-Pages zur Verfügung stellen. Grundsätzlich ist dabei zu beachten, dass das verwendete Repository öffentlich ist. Bei einem privaten Repository ist diese Funktion nicht gegeben. Somit haben wir neben dem privaten Repository (https://github.com/Wespey/Project_LF8) das öffentliche Repository (https://github.com/Wespey/Project_LF8_mkdocs) erstellt. Die Installation wurde auf einer Linux-VM (OS: LinuxMint) durchgeführt.

    2. Installation von pip und mkdocs

    Pip ist der python-integrierte Packet-Manager. Mithilfe von sudo apt-get install pip lässt sich auf der Linux-VM dieser installieren. Anschließend lässt sich mit pip mkdocs mit folgendem Befehl installieren: pip install mkdocs mkdocs-material.

    3. MKDocs-Ordnerstruktur und Theme anlegen

    MKDocs muss direkt im öffentlichen Repository deployed werden. Dazu wird zuerst mit git clone [URL des öffentlichen Repositorys]das Reporitory lokal geklont. Danach wird im geklonten Repository die Datei mkdocs.yml angelegt.

    In dieser Datei lässt sich das Layout und Theme festlegen:

    site_name: Kunde - Auftrag - IT-Installation
 nav:
   - index.md
   - Phase-1.md
   - Phase-2.md
   - Phase-3.md
   - Phase-4.md
   - Phase-5.md
   - Phase-6.md
   - Phase-7.md
   - Phase-8.md
   - Phase-9.md
 theme:
   name: material
   markdown-extensions:
     - abbr
     - admonition
     - attr_lists
     - md_in_html
     - pydownx_superfences
   features:
     - navigation.path
     - navigation.tabs
     - navigation.sections
     - toc.integrate
     - navigation.top
     - search.suggest
     - search.highlights
     - content.tabs.link
     - content.code.annotation
     - content.code.copy

   language: de
   palette:
     - scheme: default
       toggle:
         icon: material/toggle-switch-off-outline
         name: Wechsel zum dunklen Modus
       primary: deep orange
       accent: black
     - scheme: slate
       toggle:
         icon: material/toggle-switch
         name: Wechsel zum hellen Modus
       primary: deep orange
       accent: lime

    Danach ist folgende Ordnerstruktur anzulegen:

    Project_LF8_mkdocs   <Reporsitory>
mkdocs.yml
-docs           <Ordner mit Markdown-"Seiten">
index.md
Phase-1.md
Phase-2.md
Phase-3.md
Phase-4.md
Phase-5.md
Phase-6.md
Phase-7.md
Phase-8.md
Phase-9.md

    Die einzelnen Markdown-Dateien stellen in der Navigationsleiste die unterschiedlichen Seiten dar, sofern man in der mkdocs.yml entsprechende Features angegeben hat. In den Datein lässt sich nun der Inhalt in der Sprache Markdown umsetzen.

    Ist dies nun erledigt, kann man in das Projektverzeichnis mit cd <Projektverzeichnis> wechseln und den Befehl mkdocs serve ausführen. Damit wird über die Localhost-Adresse https://127.0.0.1:8000 die MKDocs-Seite angezeigt. Dies ist sehr hilfreich wenn man z. B. Änderungen tätigt und man diese ansehen möchte, bevor man sie auf gh-pages deployed.

    4. Auf gh-pages publizieren

    Im Projektverzeichnis stehend lässt sich nun mit dem Befehl mkdocs gh-deploy der Inhalt veröffentlichen. Dieser wird in der eigenen Branch "gh-pages" deployed. Mit git checkout <Branch> lässt sich aber auch die Main-Branch auswählen und dort mkdocs deployen. Danach ist die Seite unter https://<username>.github.io/<repositoryName> online verfügbar. Bei einem weiteren Deployment wird der Inhalt überschrieben, weshalb Änderungen über Github selbst, nicht empfohlen sind.
